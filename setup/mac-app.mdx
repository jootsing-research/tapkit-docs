## Step 1: Download and install

Download the TapKit Mac app from your [dashboard](https://tapkit.ai/dashboard). Open the DMG and drag TapKit to your Applications folder.

## Step 2: Sign in

When you first launch TapKit, it opens an OAuth flow in your browser. Log in with your TapKit account (or create one). After authenticating, you'll be redirected back to the app.

{/* TODO: Screenshot of the OAuth sign-in page */}

## Step 3: Grant permissions

The Mac app needs three macOS permissions to work. You'll be prompted for each one:

### Accessibility

Allows TapKit to control Switch Control and send commands to the phone. When the system dialog appears, click **Open System Settings** and enable TapKit.

{/* TODO: Screenshot of Accessibility permission dialog */}

### Automation

Allows TapKit to run macOS Shortcuts for phone communication. Click **OK** when prompted.

{/* TODO: Screenshot of Automation permission dialog */}

### Camera

Allows TapKit to capture the phone's screen via the screencast stream. Click **OK** when prompted.

{/* TODO: Screenshot of Camera permission dialog */}

<Note>
  For a deeper explanation of why each permission is needed (and what it's *not* used for), see [Security & Privacy](/security).
</Note>

## Step 4: Switch Control configuration

The app runs an automatic configuration flow that sets up the Switch Control profiles and settings needed for phone control. This happens in the background — you'll see a brief progress indicator.

{/* TODO: GIF or screenshot sequence of the automatic Switch Control config */}

<Info>
  **What's happening:** TapKit is configuring keyboard switches (Tab, Escape, Control, System) that map to Switch Control actions. It also sets scanning mode, cursor speed, and menu item ordering. You don't need to do any of this manually.
</Info>

You can verify the configuration was applied by going to **System Settings > Accessibility > Switch Control** and checking that the switches are set up:

| Key | Switch Control Action |
|-----|----------------------|
| **Tab** | Select Item |
| **Escape** | Move to Next Item |
| **Control** | Restart Scanning |
| **System** | Stop Scanning |

## Step 5: Install macOS Shortcut

The app prompts you to install a macOS Shortcut that enables communication between the Mac app and your phone. Click **Add Shortcut** when prompted.

{/* TODO: Screenshot of the Shortcut install prompt */}

<Info>

</Info>

## Step 6: Connect Mac to iPhone

Once phone setup is complete, come back to the Mac app. Your iPhone should appear in the sidebar. If it doesn't appear automatically:

1. Make sure the iPhone is still plugged in via USB
2. Both devices should be on the same Wi-Fi network
3. Bluetooth should be enabled on both devices
4. Both should be signed into the same iCloud account

When the phone appears in the sidebar, click on it to select it. The phone's screen will appear in the center panel, and you're ready to go.

## Troubleshooting

<AccordionGroup>
  <Accordion title="macOS security warning / app won't open">
    If macOS says the app is from an unidentified developer, right-click the app and choose **Open**, then click **Open** in the dialog. You only need to do this once.
  </Accordion>

  <Accordion title="Permission dialog didn't appear">
    Go to **System Settings > Privacy & Security** and manually add TapKit to the relevant permission list (Accessibility, Automation, or Camera).
  </Accordion>

  <Accordion title="Switch Control config failed">
    Try restarting the Mac app. If the automatic configuration still fails, contact [support@tapkit.ai](mailto:support@tapkit.ai) with your macOS version and any error messages you see.
  </Accordion>

  <Accordion title="Shortcut won't install">
    Make sure you're running macOS 12 (Monterey) or later. If the Shortcuts app isn't installed, you can get it from the Mac App Store. Try restarting the Mac app and clicking the install prompt again.
  </Accordion>

  <Accordion title="Phone not appearing after connection">
    Verify both devices are on the same Wi-Fi network, Bluetooth is enabled, and both are signed into the same iCloud account. Try unplugging and replugging the USB cable. If you're on a public Wi-Fi network (coffee shop, hotel), try switching to a personal hotspot.
  </Accordion>

  <Accordion title="Wi-Fi / Bonjour / network issues">
    TapKit uses Apple's Bonjour protocol for device discovery, which requires both devices on the same local network. Public or enterprise Wi-Fi networks that isolate devices will prevent discovery. Use a home network or personal hotspot instead.
  </Accordion>
</AccordionGroup>

If you're experiencing issues not covered here, [contact our support team](mailto:support@tapkit.ai).
